ENDLOOP.

  WRITE:'no data'.

[김문영] [오후 6:45] # Class
 - T-Code:SE24 전역, 지역 클래스 정의
 - 프로그램 내부의 정의된 지역클래스를 먼저 탐색 후 없으면 전역 클래스 탐색
 - DFINITION : 클래스의 구성항목(Attributes, Method,Events)을 정의한다.
 - IMPLEMENTATION : 클래스의 Method기능을 구현
 *> 클래스의 객체마다 존재
 - Instance Component(Instance), 객체를 생성할 때 마다 초기화
 *> 전체 클래스에서 오직 하나만 존재
 - Static Component(Non Instance), 
생성자(CREATE OBJECR 구문)를 만나면 프로그램이 종료 할 때 까지 메모리에 저장
 - 객체를 생성하지 않아도 메모리에 로드되어 바로 사용 가능
 
 *) Attribute 
* 모든 ABAP 데이터 타입을 가질 수 있는 클래스의 내무 데이터 필드
 * Reference Variable은 객체를 생성하고 주소를 지정
 * 클래스 내부에서 정의 될 수 있으며, 클래스 내부로부터 객체에 접근할 수 있도록 해준다.
 * Instance Attribute
 - 클래스 내에서 DATA구문을 사용해서 인스턴스를 선언
 - 객체를 생성할 때마다 초기화됨
 * Static Attribute
 - CLASS-DATA 구문을 이용해서 선언
 - 객체를 생성할 때마다 초기화되지 않음
 
 *) Method
 * 객체의 행위를 정의하는 클래스의 내부 수행 절차를 정의
 * 클래스의 PRIVATE속성은 동일 클래스의 메소드에 의해서만 변경
 * 정의(DEFINITION)부부에서 METHOD 구문을 이용해 정의
 * IMPLEMENTATION 부분에서 METHOD 기능을 기술
 * Instance Method
 - METHODS 구문으로 Instance Method 선언
 * Static Method
 - CLASS_METHODS 구문으로 Static Method 선언
 * Special Method
 - CONSTRUCTOR, CLASS-CONSTRUCTOR, EVENTS
 
 *) Event
 * 상속 관계에 있지 않은, 클래스 간에 메소드를 상호 호출하여 영향을 미칠 수 있는 특별한 메소드
 * 이벤트 핸들러 메소드를 호출하기 위해 이벤트를 사용
 * 모든 객체들이 이벤트 메소드를 호출할 수 있다.
 * 이벤트의 소유주인 객채만이 이벤트를 실행할 수 있다.
 
 *) 접근제한 영역
 * PUBLIC : 모든 클래스에서 접근이 가능
 * PROTECTED : 자신과 상속받은 클래스에서만 접근 가능
 * PRIVATE : 동일한 클래스의 메소드에서만 접근 가능, 외부에서 저븐 불가, 클래스 내부에서 캡슐화

Lock 설정 순서
(1) Requests Lock : 프로그램에서 데이터에 대한 Lock을 요청한다. 잠긴(Locked) 상태의 테이블 데이터 레코드는 논리적인 조건(logical condition)에 의해 관리된다.
(2) Endter Lock : Lock 데이터의 조건이 Lock Table에 입력된다.
(3) Set Lock : 프로그램 요청에 의해 Lock이 설정된다. Lock 상태는 프로그램이 Unlock을 수행하거나 프로그램이 종료되는 시점까지 유지된다. 모든 Lock은 프로그램에 의해 설정되므로 해당 프로그램에서 해지해야 한다. 데이터가 잠긴 상태에서 다른 프로그램에서 데이터에 접근하면 이 요청은 Lock Table에 저장된다. 작업하고자 하는 데이터를 정확하게 설정하기 위해, Lock Object는 테이블의 Key 필드로 구분될 수 있도록 설정하는 것이 바람직하다.
(4) Access Data : lock을 설정한 프로그램에서만 해당 데이터에 접근하여 작업할 수 있다.
